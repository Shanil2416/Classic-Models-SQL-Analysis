*Classic Models Database Analysis Using SQL*

# Project Overview
This project analyzes the Classic Models sample database using SQL to answer real-world business questions. The analysis focuses on customers, orders, products, payments, employees, and sales performance to generate meaningful business insights.

# Domain
Sales Analytics

# Tools Used
- MySQL
- MySQL Workbench
- SQL

# Dataset
Classic Models Sample Database (MySQL)

# Problem Statement
Analyze the Classic Models database using SQL to understand customer behavior, sales performance, product demand, employee performance, and payment trends to support business decision-making.

# SQL Concepts Used
- SELECT
- WHERE
- ORDER BY
- GROUP BY
- HAVING
- JOINS
- Aggregate Functions
- Subqueries
- CTEs
- Window Functions
- String Functions
- Date Functions
- UPDATE
- DELETE

# Project Workflow
1. Imported the Classic Models database into MySQL.
2. Explored tables and relationships.
3. Wrote SQL queries to answer 50 business questions.
4. Performed customer, sales, payment, employee, and product analysis.
5. Generated business insights from query results.

# Business Areas Covered
- Customer Analysis
- Sales Analysis
- Product Analysis
- Order Analysis
- Payment Analysis
- Employee Analysis
- Office Analysis

# Key Insights
- Identified top customers based on payment amount.
- Analyzed product line performance.
- Compared customer payments across countries.
- Found products with no sales.
- Identified high-value orders and payments.
- Evaluated employee and office performance.
- Generated insights for business decision-making.
